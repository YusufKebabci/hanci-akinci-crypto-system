BÖLÜM 5 — AES ANAHTARI (K) ÜRETİMİ  

AES-CTR şifrelemesinde her iki taraf da aynı anahtarı kullanır. Bu anahtarın güvenli
şekilde paylaşılması gerekir. KEY olarak adlandırılan AES anahtarı 32 byte yani 256 bit
uzunluğunda, openssl rand ile kriptografik olarak güvenli biçimde üretiliyor. IV olarak
adlandırılan başlangıç vektörü 16 byte uzunluğunda olup her şifreleme işlemi için farklı
bir başlangıç noktası belirliyor. Her iki değer anahtar.txt dosyasına kaydediliyor.
Bu dosya HANCI tarafında üretiliyor ve AKINCI'ya iletiliyor.

HANCI

AES_KEY=$(openssl rand -hex 32)
AES_IV=$(openssl rand -hex 16)

echo "AES Anahtarı (K) : $AES_KEY"
echo "AES IV (Nonce)   : $AES_IV"

echo "KEY=$AES_KEY" > anahtar.txt
echo "IV=$AES_IV"  >> anahtar.txt
HANCI
cat anahtar.txt

cp anahtar.txt ../AKINCI/
echo "Anahtar AKINCI'ya iletildi."


BÖLÜM 6 — LOOKUP TABLOSU 

AKINCI tarafında 800.000 satırlık SHA-256 arama tablosu oluşturuluyor. ZIP dosyasının
byte boyutunu şifreli pakete direkt yazmak yerine hash'ini koyuyoruz. Alıcı bu hash'i
tabloda aratarak orijinal boyutu buluyor. Tablo formatı her satırda SAYI:HASH şeklinde.
Bu işlemi Python ile yapıyoruz çünkü saf Bash döngüsüyle her sayı için ayrı bir süreç
başlatılırdı ve saatlerce sürerdi. Python'da tek süreçte 2-3 dakikada tamamlanıyor.

AKINCI

python3 -c "
import hashlib
with open('boyut.txt', 'w') as f:
    for i in range(1, 800001):
        h = hashlib.sha256(str(i).encode()).hexdigest()
        f.write(f'{i}:{h}\n')
        if i % 100000 == 0:
            print(f'  {i}/800000 tamamlandı...')
print('Lookup tablosu hazır.')
"

wc -l boyut.txt
head -3 boyut.txt

BÖLÜM 8 — 500x500 BMP İŞLEMLERİ

GIMP ile hazırlanan 500x500.bmp HANCI klasörüne alınıyor. GIMP dosyaya ekstra renk profili
(ICC metadata) eklediğinden header boyutu standart 54 byte yerine 138 byte oluyor. Bunu
bilmeden 54 byte kessek body bozulurdu. Python ile BMP dosyasının 10-14. byte'ları okunuyor,
bu alanda piksel verisinin nerede başladığı yazıyor ve OFFSET değişkenine atanıyor. Bu
değer dd ve tail komutlarında kullanılıyor. Dört farklı hash üretiliyor: header hash
başlığın, body hash gövdenin, ZIP hash sıkıştırılmış verinin doğruluğunu, boyut hash ise
lookup tablosunda boyutu bulmayı sağlıyor. Header ve hash dosyaları AKINCI'ya kopyalanıyor.

HANCI

cp ~/Documents/500x500.bmp .
wc -c 500x500.bmp
file 500x500.bmp

OFFSET=$(python3 -c "
import struct
with open('500x500.bmp','rb') as f:
    data = f.read()
print(struct.unpack('<I', data[10:14])[0])
")
echo "Header offset: $OFFSET byte"

dd if=500x500.bmp of=500x500header.bin bs=1 count=$OFFSET 2>/dev/null
tail -c +$(( OFFSET + 1 )) 500x500.bmp > 500x500body.bin
echo "500x500header.bin : $(wc -c < 500x500header.bin) byte"
echo "500x500body.bin   : $(wc -c < 500x500body.bin) byte"

sha256sum 500x500header.bin | awk '{printf "%s", $1}' > 500x500headerhash.txt
echo "Header hash: $(cat 500x500headerhash.txt)"

cp plainheader1.bin      ../AKINCI/
cp plainheader2.bin      ../AKINCI/
cp plainheader3.bin      ../AKINCI/
cp plainheader4.bin      ../AKINCI/
cp plainheader5.bin      ../AKINCI/
cp plainheader1hash.txt  ../AKINCI/
cp plainheader2hash.txt  ../AKINCI/
cp plainheader3hash.txt  ../AKINCI/
cp plainheader4hash.txt  ../AKINCI/
cp plainheader5hash.txt  ../AKINCI/
cp 500x500header.bin     ../AKINCI/
cp 500x500headerhash.txt ../AKINCI/
echo "Kopyalama tamamlandı"

sha256sum 500x500body.bin | awk '{printf "%s", $1}' > 500x500bodyhash.txt

echo "Boyut hash: $(cat boyuthash.txt)"


BÖLÜM 9 — MAC PAKETİ, LCG DOLGU VE AES-CTR 

Dört hash değeri ve ZIP binary verisi birleştirilerek payload oluşturuluyor. MAC paketinin
ilk 256 byte'ı dört hash bloğundan oluşuyor: 0-63 header hash, 64-127 body hash, 128-191
ZIP hash, 192-255 boyut hash. 256. byte'tan itibaren ZIP binary verisi geliyor. En sona
LCG dolgusu ekleniyor. LCG formülü şu: X(n+1) = (1664525 × Xn + 1013904223) mod 2^32.
Tohum değeri 1011. Paket boyutunu 16'nın katına tamamlamak için kullanılıyor. Print
satırları stderr'a yönlendiriliyor çünkü stdout'a gitseydi binary dosyaya karışırdı ve
dosya bozulurdu. AES şifrelemesinde K ve IV anahtar.txt'den okunuyor. Cipher BMP
oluştururken sadece AES-CTR dosyası konulsaydı resmin büyük kısmı siyah kalır ve şifreli
kısmın boyutu tahmin edilebilirdi. Bunu önlemek için AES-CTR dosyasının arkası LCG
verileriyle body boyutuna tamamlanıyor. Böylece resmin tamamı şifreli görünüyor.

HANCI

cat \
  500x500headerhash.txt \
  500x500bodyhash.txt   \
  500x500ziphash.txt    \
  boyuthash.txt         \
  500x500body.zip       \
  > payload.bin
echo "Payload: $(wc -c < payload.bin) byte"

PAYLOAD_BOYUT=$(wc -c < payload.bin)

python3 -c "
import sys
payload  = int(sys.argv[1])
kalan    = payload % 16
hizalama = (16 - kalan) % 16
toplam   = 16 + hizalama
print(f'Payload boyutu : {payload} byte',           file=sys.stderr)
print(f'Toplam LCG     : {toplam} byte',            file=sys.stderr)
print(f'Sonuç boyutu   : {payload + toplam} byte',  file=sys.stderr)
print(f'16nın katı mı  : {(payload+toplam)%16==0}', file=sys.stderr)
seed, a, c, m = 1011, 1664525, 1013904223, 2**32
buf, x = bytearray(), seed
for _ in range(toplam):
    x = (a * x + c) % m
    buf.append(x & 0xFF)
sys.stdout.buffer.write(bytes(buf))
" "$PAYLOAD_BOYUT" > lcg_dolgu.bin

cat payload.bin lcg_dolgu.bin > 500x500mac.bin
echo "MAC boyutu: $(wc -c < 500x500mac.bin) byte"
python3 -c "b=$(wc -c < 500x500mac.bin); print(f'16nın katı mı: {b%16==0}')"

AES_KEY=$(grep "KEY=" anahtar.txt | cut -d= -f2)
AES_IV=$(grep "IV="  anahtar.txt | cut -d= -f2)
echo "Kullanılan K  : $AES_KEY"
echo "Kullanılan IV : $AES_IV"

openssl enc -aes-256-ctr \
  -nosalt -nopad \
  -K  "$AES_KEY" \
  -iv "$AES_IV"  \
  -in  500x500mac.bin \
  -out 500x500AES-CTR.bin
echo "AES-CTR boyutu: $(wc -c < 500x500AES-CTR.bin) byte"

BODY_BOYUT=$(wc -c < 500x500body.bin)
AES_BOYUT=$(wc -c < 500x500AES-CTR.bin)
KALAN=$(( BODY_BOYUT - AES_BOYUT ))
echo "Doldurulacak: $KALAN byte"

python3 -c "
import sys
n = int(sys.argv[1])
seed, a, c, m = 1011, 1664525, 1013904223, 2**32
buf, x = bytearray(), seed
for _ in range(n):
    x = (a * x + c) % m
    buf.append(x & 0xFF)
sys.stdout.buffer.write(bytes(buf))
" "$KALAN" > goruntu_dolgu.bin

cat 500x500header.bin \
    500x500AES-CTR.bin \
    goruntu_dolgu.bin \
    > 500x500chiper.bmp
echo "Cipher BMP : $(wc -c < 500x500chiper.bmp) byte"
echo "Orijinal   : $(wc -c < 500x500.bmp) byte"

eog 500x500chiper.bmp

BÖLÜM 11 — AKINCI: DOĞRULAMA VE GERİ KAZANIM

AKINCI tarafı gelen şifreli dosyayı katman katman çözüyor ve her adımda doğrulama
yapıyor. İlk olarak GPG şifresi çözülüyor ve HANCI'nın imzası doğrulanıyor. Terminalde
"Good signature from HANCI" görünmesi gerekiyor. Ardından AES-CTR şifresi çözülüyor ve
MAC paketine ulaşılıyor. dd komutuyla her biri 64 byte olan dört hash bloğu ayrıştırılıyor.
Header hash doğrulamasında cmp komutu iki dosyayı bit bit karşılaştırıyor. Boyut hash'i
grep ile boyut.txt'de aranıyor, satır sonuna dolar işareti konularak tam eşleşme sağlanıyor
ve ZIP'in orijinal boyutu elde ediliyor. Bu boyut bilgisiyle dd komutu 256. byte'tan
itibaren ZIP verisini ayırıyor, LCG dolgusu geride kalıyor ve atılıyor. ZIP doğrulanıyor,
açılıyor, body doğrulanıyor ve son olarak header ile body birleştirilip orijinal BMP
geri kazanılıyor.

AKINCI

gpg --decrypt \
    --output 500x500AES-CTR.bin \
    500x500AES-CTR.bin.gpg

AES_KEY=$(grep "KEY=" anahtar.txt | cut -d= -f2)
AES_IV=$(grep "IV="  anahtar.txt | cut -d= -f2)
echo "Kullanılan K  : $AES_KEY"
echo "Kullanılan IV : $AES_IV"

openssl enc -d -aes-256-ctr \
  -nosalt -nopad \
  -K  "$AES_KEY" \
  -iv "$AES_IV"  \
  -in  500x500AES-CTR.bin \
  -out 500x500mac.bin
echo "MAC boyutu: $(wc -c < 500x500mac.bin) byte"

dd if=500x500mac.bin bs=1 skip=0   count=64 of=500x500headercheck.txt 2>/dev/null
echo "1. Header check: $(cat 500x500headercheck.txt)"
dd if=500x500mac.bin bs=1 skip=64  count=64 of=500x500bodycheck.txt   2>/dev/null
echo "2. Body check  : $(cat 500x500bodycheck.txt)"
dd if=500x500mac.bin bs=1 skip=128 count=64 of=500x500zipcheck.txt    2>/dev/null
echo "3. ZIP check   : $(cat 500x500zipcheck.txt)"
dd if=500x500mac.bin bs=1 skip=192 count=64 of=boyuthash_check.txt    2>/dev/null
echo "4. Boyut check : $(cat boyuthash_check.txt)"

cmp -s 500x500headercheck.txt 500x500headerhash.txt \
  && echo "✔ HEADER DOĞRULANDI" \
  || echo "✖ HATA"

ARANAN=$(cat boyuthash_check.txt)
echo "Aranan hash: $ARANAN"
ZIP_BOYUT=$(grep -m1 ":${ARANAN}$" boyut.txt | cut -d: -f1)
echo "✔ ZIP boyutu bulundu: $ZIP_BOYUT byte"

dd if=500x500mac.bin bs=1 skip=256 count=$ZIP_BOYUT \
   of=500x500body.zip 2>/dev/null
echo "Ayrılan ZIP: $(wc -c < 500x500body.zip) byte"

HESAPLANAN=$(sha256sum 500x500body.zip | awk '{printf "%s", $1}')
BEKLENEN=$(cat 500x500zipcheck.txt)
[ "$HESAPLANAN" = "$BEKLENEN" ] \
  && echo "✔ ZIP DOĞRULANDI" \
  || echo "✖ HATA"

unzip -o 500x500body.zip -d .
echo "Çıkarılan: $(ls -lh 500x500body.bin)"

HESAPLANAN=$(sha256sum 500x500body.bin | awk '{printf "%s", $1}')
BEKLENEN=$(cat 500x500bodycheck.txt)
[ "$HESAPLANAN" = "$BEKLENEN" ] \
  && echo "✔ BODY DOĞRULANDI" \
  || echo "✖ HATA"

cat 500x500header.bin 500x500body.bin > 500x500_recovered.bmp
echo "Recovered BMP: $(wc -c < 500x500_recovered.bmp) byte"

cmp -s ../HANCI/500x500.bmp 500x500_recovered.bmp \
  && echo "✔ MÜKEMMEL — Orijinal ile birebir aynı!" \
  || echo "✖ Dosyalar farklı!"

eog 500x500_recovered.bmp
